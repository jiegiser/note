# JavaScript 数据类型以及判断

JavaScript 有 7 种数据类型，分别是 null、undefined、boolean、number、string、object、symbol（ES6）。前 5 种是基本类型数据，object 类型又具体包含了 function、Array、date 等；

对于类型推断，常用的方法有 typeof、instanceof、Object.prototype.toString、constructor；

# 使用 typeof 判断数据类型

基本数据类型可以使用 typeof 来判断：

typeof 5 // 'number'
typeof 'jie' // 'string'
typeof undefined // 'undefined'
typeof true // 'boolean'

// 特殊的
typeof null // 'object'

typeof 判断复杂类型：

const foo = () => 1
typeof foo // 'function'

const foo = {}
typeof foo // 'object'

const foo = []
typeof foo // 'object'

const foo = new Date()
typeof foo // 'object'

const foo = Symbol('foo')
typeof foo // 'symbol'

使用 typeof 可以准确判断出除 null 以外的基本类型数据，以及 function 类型，symbol 类型；null 会被判断为 object；

# 使用 instanceof 判断数据类型

用法： a instanceof, a 是否为 B 的实例；也就是说 a 的原型链上是否存在 B 的构造函数；

关于 instanceof 的原理，可以用下面的代码来模拟：

// L 表示左表达式，R 表示右表达式
const instanceofMock = (L, R) => {
  if (typeof L !== 'object') {
    return false
  }
  while (true) {
    if (L === null) {
      // 已经遍历到最顶端
      return false
    }
    if (R.prototype === L.__proto__) {
      return true
    }
    L = L.__proto__
  }
}

// 使用
function Person(name) {
  this.name = name
}

const p = new Person('jie')
instanceofMock(p, Person) // true

# 使用 constructor 和 Object.prototype.toString 判断数据类型

在判断数据类型时，我们称 Object.prototype.toString 为万能方法、终极方法，示例代码：

console.log(Object.prototype.toString.call(1)) // [object Number]

console.log(Object.prototype.toStirng.call('jie')) // [object String]

console.log(Object.prototype.toString.call(undefined)) // [object Undefined]

console.log(Object.prototype.toString.call(true)) // [object Boolean]

console.log(Object.prototype.toString.call([])) // [object Array]

console.log(Object.prototype.toString.call(function() {})) // [object Function]

console.log(Object.prototype.toString.call(null)) // [object Null]

console.log(Object.prototype.toString.call(Symbol('jie'))) // [object Symbol]

使用 constructor 可以查看目标的构造函数，也可以进行数据类型判断，但其中也存在问题，如下代码：

var foo = 5
foo.constructor // f Number() { [native code] }

var foo = 'jie'
foo.constructor // ƒ String() { [native code] }

var foo = true
foo.constructor // ƒ Boolean() { [native code] }

var foo = []
foo.constructor // ƒ Array() { [native code] }

var foo = {}
foo.constructor // ƒ Object() { [native code] }

var foo = () => 1
foo.constructor // ƒ Function() { [native code] }

var foo = new Date()
foo.constructor // ƒ Date() { [native code] }

var foo = Symbol('foo')
foo.constructor // ƒ Symbol() { [native code] }

var foo = undefined
foo.constructor
// VM1830:2 Uncaught TypeError: Cannot read property 'constructor' of undefined

    // at <anonymous>:3:5

对于 null、undefined 会报错；constructor 返回的是构造函数本身。

# JavaScript 数据类型及其转换

# + 运算符数据类型的转换

  • 基本类型

当使用 + 运算符计算 string 类型和其他数据类型相加时，其他数据类型都会转换为 string 类型；而在其他情况下，都会转换为 number 类型，但是 undefined 会转换为 NaN，相加结果也是 NaN；

  • 复杂类型

在 + 运算符两侧，如果存在复杂数据类型，比如对象，那么他将会被转换为基本数据类型再进行运算。在转换时，会调用该对象上的 valueOf 或 toString 方法。这两个方法的返回值就是转换后的结果。

valueOf 以及 toString 是可以被开发者重写的：

const foo = {
  toString() {
    return 'jie'
  },
  valueOf() {
    return 1
  }
}

当使用 alert 打印该对象时，会输出 jie，他会倾向于使用对象的 toString 方法，而使用 console.log(1 + foo) 将输出2，这时候的隐式转换则倾向于使用 foo 对象的 valueOf 方法。

对于加法操作，如果 + 运算符两边都是 number 类型，则其规则如下：

  • 如果 + 运算符两边存在 NaN，则结果为 NaN（typeof NaN 为 number）
  • 如果是 Infinity + Infinity，则结果是 Infinity
  • 如果是 -Infinity + (-Infinity)，则结果是 -Infinity
  • 如果是 -Infinity + Infinity，则结果是 NaN

如果 + 运算符两边有至少一个是字符串，则其规则如下：

  • 如果两边都是字符串，则执行字符串的拼接操作
  • 如果两边只有一个是字符串，则将另外的值转换为字符串，再执行字符串拼接操作
  • 如果两边有一个是对象，则调用 valueOf 或 toString 方法取得值，将其转换为基本数据类型再进行字符串拼接

# JavaScript 函数参数传递

  • 函数参数为基本数据类型时，函数体内会复制一份参数值，对此修改不会影响原参数的实际值
  • 函数参数是引用类型时，挡在函数体内修改了参数的某个属性，会修改原来的参数对应的实际值
  • 函数参数是引用类型时，如果直接修改这个值的引用地址，则相当于在函数体内心创建了一个引用，任何操作都不会影像原参数的实际值

# cannot read prototype of undefined 问题解决方案

对于下面代码：

const obj = {
  user: {
    posts: [
      { title: 'Foo', comments: [ 'good one!', 'interesting...' ] },
      { title: 'Bar', comments: [ 'ok' ] },
      { title: 'Baz', comments: [] }
    ],
    comments: []
  }
}
  1. 通过 && 短路运算符进行可访问性嗅探
obj.user &&
obj.user.posts &&
obj.user.posts[0] &&
obj.user.posts[0].comments
  1. 通过 || 单元设置默认保底值
(((obj.user || {}).posts || {})[0] || {}).comments
  1. 使用 try...catch 方法
var result
try {
  result = obj.user.posts[0].comments
} catch {
  result = null
}
  1. 使用 lodash get API
function get(object, path, defaultValue) {
  const result = object === null ? undefined : baseGet(object, path)
  return result === undefined ? defaultValue : result
}

function baseGet(object, path) {
  path = castPath(path, object)

  let index = 0
  while(object != null && index < length) {
    object = object[toKey(path[index++])]
  }
  return (index && index == length) ? object : undefined
}

自己也可以实现一个简易的 get。代码如下：

const get = (p, o) => p.reduce((xs, x) => (xs && xs[x]) ? xs[x] : null , o)

console.log(get(['user', 'posts', 0, 'components'], obj)) // [ 'good one!', 'interesting...' ]

同样可以对方法进行柯里化：

const get = p => o =? p.reduce((xs, x) => xs && xs[x] ? xs[x] : null, o)

const getUserComponents = get(['user', 'posts', 0, 'components'])

console.log(getUserComponents(obj))
