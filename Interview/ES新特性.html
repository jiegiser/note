# Object spread 和 Object.assign

Object spread 和 Object.assign 在很多情况下做的事情是一致的，但是他们之间一定还存在区别。实际上 Object.assign 将会修改他的第一个参数对象，这个修改可以触发其第一个参数对象的 setter。在函数编程中，比如 React、Redux 都是有不可变性的概念。从这层面上来说，Object spread 操作符会创建一个对象的副本，而不会修改任何值，在对不变性有要求的情况下是一种最好的选择。

当然 Object.assign 的第一个参数为空对象时，也可以实现同样的 “不变性”，但是这样性能就比 Object spread 差太多。使用 Object spread 的性能要明显领先于使用 Object.assign 。

Object.assign 只是复制对象的可枚举属性，不适合深拷贝。

# 箭头函数不适用的场景

构造函数的原型方法需要通过 this 获得实例，因此箭头函数不可以出现在构造函数的原型方法上，下面例子就是错误用法：

Persion.prototype = () => {
  // ...
}

需要获得 arguments 时，箭头函数不具有 arguments 属性，因此在其函数体内无法访问这一特殊的伪数组。所以不适用箭头函数。

同样在定义一个对象方法时，预期在方法中能通过 this 访问到对象实例的代码如下：

const person = {
  name: 'lucas',
  getName: () => {
    console.log(this.name)
  }
}
person.getName()

在上面代码中 getName 函数体内的 this 指向的是 window，不适用箭头函数。

同理，在使用动态回调时，对于类似下面这种对回调函数的 this 有特殊场景需求的用法，箭头函数的 this 也是无法满足：

const btn = document.getElementById('btn')
btn.addEventListener('click', () => {
  console.log(this === window) // true
})

# Proxy

一个简单的例子：

class Person {
  constructor(name) {
    this.name = name
  }
}

// 对 Person 构造函数进行了代理，这样就可以防止他作为非构造函数被调用
let proxyPersonClass = new Proxy(Person, {
  apply(target, context, args) {
    throw new Error(`hello: Function ${target.name} cannot be invoked without 'new'`)
  }
})

proxyPersonClass('lucas')
// VM43:10 Uncaught Error: hello: Function Person cannot be invoked without 'new'
// at Object.apply (<anonymous>:10:11)
// at <anonymous>:14:

new proxyPersonClass('lucas')
// { name: 'lucas' }

同样也可以使用代理，将被非构造函数调用的函数处理成强制使用 new 调用：

let proxyPersonClass = new Proxy(Person, {
  apply(target, context, args) {
    return new (target.bind(context, ...args))
  }
})

// 这样即便不适用 new 关键字时，仍然可以得到 new 调用的实例
ProxyPersonClass('lucas')
// { name: 'lucas' }

在前端测试中有 assert 断言，一种常用的方式如下：

const lucas = {
  age: 23
}

// 当 assert 右侧表达式不成立时，断言失败，断言就会抛出错误
assert['lucas is older than 22!!'] = 22 > lucas.age
// Error: lucas is older than 22 !!

其实断言库本质上还是拦截 assert 对象的赋值 set 操作，具体代码如下：

// 判断对 assert 的赋值是否为 true，如果不为 true，则打印错误
const assert = new Proxy({}, {
  set(target, context, value) {
    if (!value) {
      console.error(warning)
    }
  }
})

# Decorator

装饰器让你可以在设计时对类和类的属性进行注解和修改。 我们知道，在对象方法中给普通函数赋值后，该普通函数在全局作用域下被调用时，this 的指向会丢失，代码如下：

class Person {
  constructor(name) {
    this.name = name
  }
  getPerosnName() {
    return this.name
  }
}

const person = new Person('lucas')
const fn = person.getPerosnName

fn()
// VM998:6 Uncaught TypeError: Cannot read property 'name' of undefined
    // at getPerosnName (<anonymous>:6:17)
    // at <anonymous>:13:1

在执行 fn 的时候，this 已经指向了 window，使用 autobind 库可以完成对 this 的绑定，或者可以使用箭头函数；如下代码：

class Person {
  constructor(name) {
    this.name = name
  }
  @autobind
  getPerosnName() {
    return this.name
  }
}

autobind 实现的伪代码如下：

/**
  target: 目标对象，这里作用于 Person 中的函数、属性
  key: 属性名称
  descriptor: 属性原本的描述符
*/
function autobind(target, key, { value: fn, configurable, enumerable }) {
  return {
    configurable,
    enumerable,
    get() {
      const boundFn = fn.bind(this)
      defineProperty(this, key, {
        configurable: true,
        writable: true,
        enumerbale: false,
        value: boundFn
      });
      return boundFn;
    },
    set: createDefaultSetter(key)
  }
}

autobind 这个装饰器函数最终会返回描述符，这个描述符运行时相当于调用 Object.defineProperty() 修改原有属性，最终的修改结果如下：

{
  configurable,
  enumerable,
  get() {
    const boundFn = fn.bind(this)
    defineProperty(this, key, {
      configurable: true,
      writable: true,
      enumerable: false,
      value: boundFn
    })
    return boundFn
  },
  set: createDefaultSetter(key)
}

# const、let 编译分析

这两个编译后都是 var，为了保证 const 的不变性，Babel 如果在编译过程中发现对 const 声明的变量进行了二次赋值，则会直接报错，这样就可以在编译阶段对错误进行处理。至于 let 的块级作用域概念，我们在 ES5 中一般通过 IIFE（立即调用函数表达式）实现块级作用域，Babel 的处理就是在块内给变量换一个名字，这样在块外自然就无法被访问到了。

对于 let、const 的暂时性死区，Babel 是将 let、const 变量重新命名后，同时开启 strict mode 模式。严格模式下不允许使用未声明的变量。

对于 const 声明的变量，其变量（内存地址）是不可改变的，Babel 对于 const 声明的变量，都会插入一个 _readOnlyError() 函数。这个函数的执行内容就是报错。

# 箭头函数的编译分析

一个例子：

var obj = {
  prop: 1,
  func: function() {
    var _this: this;

    var innerFunc: () => {
      this.prop = 1
    };

    var innerFunc1 = function() {
      this.prop = 1
    };
  }
}

Babel 会将其转换为下面代码：

var obj = {
  prop: 1,
  func: function func() {
    var _this2 = this;
    var _this = this;
    var innerFunc = function innerFunc() {
      _this2.prop = 1
    }
    var innerFunc1 = function innerFunc1() {
      this.prop = 1
    }
  }
}
