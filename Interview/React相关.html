# this.setState

setState 方法并不总是能够立即更新组件，他可能会延迟更新，也不总是同步执行。所谓的延迟更新并不是针对所有情况的。

实际上，在 React 控制之内的事件处理过程中，setState 不会同步更新 this.state；而对于 React 控制之外的情况，setState 会同步更新 this.state。

如下例子：

onclick() {
  this.setState({
    count: this.state.count + 1
  })
}

componentDidMount() {
  document.querySelectorAll('#btn-raw').addEventListener('click', this.onClick)
}

render() {
  return (
    <React.Fragment>
      <button id="btn-raw">
        click out React
      </button>

      <button onClick={this.onClick}>
        click in React
      </button>
    </React.Fragment>
  )
}

id 为 btn-raw 的 button 上绑定的事件，是在 componentDidMount 方法中通过 addEventListener 完成更新的，这是脱离于 React 事件之外的事件；因此他会同步更新。

延迟更新的目的也是为了获得更好的性能。

# 原生事件和 React 合成事件

React 中的事件机制并不是原生的那一套，事件没有绑定在原生 DOM 上，大多数事件都绑定在 document 上，除了少数不会冒泡到 document 的事件，如 video 等。同时 React 触发的事件也是对原生事件的包装，并不是原生 event 对象。

合成事件会引发下面几个潜在的问题：

# 异步访问事件对象

不能以异步的方式访问合成事件对象，如下错误的例子：

function handleClick(e) {
  console.log(e)

  setTimeout(() => {
    // undefined
    console.log(e)
  }, 0)
}

React 也有持久化合成事件的方法，如下：

function handleClick(e) {
  console.log(e)

  e.persist()

  setTimeout(() => {
    console.log(e)
  }, 0)
}

# 阻止原生事件冒泡

在 React 中直接使用 e.stopPropagation 不能阻止原生事件冒泡。因为事件早已经冒泡到了 document 上。React 在事件冒泡到 document 上时才能够处理事件。如下代码，是不能阻止事件冒泡的：

componentDidMount() {
  document.addEventListener('click', () => {
    console.log('document click')
  })
}

handleClick = e => {
  console.log('div click')
  e.stopPropagation()
}

render() {
  return (
    <div onClick = {this.handleClick}>
      click
    </div>
  )
}

上面代码首先打印 div click，然后打印 document click。没有阻止冒泡。

但是 React 的合成事件为使用原生事件留了一个口子，通过合成事件上的 nativeEvent 属性可以访问原生事件。原生事件上的 stopImmediatePropagation 方法除了能够像 stopPropagation 一样阻止事件冒泡，还能阻止当前元素剩余的、同类型事件的执行（第一个 click 事件触发时，调用 e.stopImmediatePropagation 能够阻止当前元素的第二个 click 事件的触发）

如下代码：

componentDidMount() {
  document.addEventListener('click', () => {
    console.log('document click')
  })
}

handleClick = e => {
  console.log('div click')
  e.nativeEvent.stopImmediatePropagation()
}

render() {
  return (
    <div onClick = {this.handleClick}>
      click
    </div>
  )
}

# 组件通讯和封装

class App extends Component {
  constructor(props) {
    super(props)
    this.state = { number: 0 }
  }

  updateNumber(toAdd) {
    this.setState(prevState => {
      number: prevState.number + toAdd
    })
  }

  render() {
    return (
      <div className="app">
        <span className="number">{this.state.number}</span>
        <Controls
          onIncrease={() => this.updateNumber(+1)}
          onDecrease={() => this.updateNumber(-1)}
        />
      </div>
    )
  }
}


const Controls = ({ onIncrease, onDecrease }) => {
  return (
    <div className="controls">
      <button onClick={onIncrease}>Increase</button>
      <button onClick={onDecrease}>Decrease</button>
    </div>
  )
}

# 副作用和纯组件

首先纯函数是通过函数参数能够唯一确定函数返回值的函数。

准纯组件是渲染数据全部来自 props，但是会产生副作用的组件，下面是一段非纯组件的代码：

const globalConfig = {
  siteName: 'Animals in Zoo'
}

const Header = ({ children }) => {
  const heading = globalConfig.siteName ? <h1>{globalConfig.siteName}</h1> : null

  return (
    <div>
      {heading}
      {children}
    </div>
  )
}

上面这个组件是典型的非纯组件，因为它依赖全局变量 siteName，其可能渲染出两种结果。

可以将上面的组件进行优化，使全局变量作为 Header 的 props 出现，而不再是一个外部的全局变量，这样依赖，函数式组件 Header 就完全依赖其参数来决定渲染内容了，代码如下：

const globalConfig = {
  siteName: 'Animals in Zoo'
}

const Header = ({ children, siteName }) => {
  const heading = siteName ? <h1>{siteName}</h1> : null

  return (
    <div>
      {heading}
      {children}
    </div>
  )
}

Header.defaultProps = {
  siteName: globalConfig.siteName
}

对于数据状态管理，Redux 提倡以函数式、不可变性、数据扁平化的形式进行；而其中的获取数据说到底是依赖发布/订阅模式进行的。相对地，Mobox 是面向对象和响应式的结合，他的数据源时可变的，对数据的观察是响应式的。

# React Component

class 声明组件的问题有以下两个问题：

  • 带来了面向声明周期编程的困扰，随着逻辑变复杂，组件的声明周期函数随之变得很难维护和理解。我们想弄清楚 componentDitMount、componentDitUpdate、componentWillUnmount、componentWillReceiveProps 这些钩子的逻辑并不困难，但是这些声明周期函数中的代码和 render 中的 state 及 props 有什么关系，这种关系将会随着应用越来越复杂被无限放大。

  • react 是函数式的，而 class 声明组件这种面相对象的行为显得不伦不类。

# hooks

useState 模拟实现代码：

const react = (function() {
  let stateValue

  return Object.assign(React, {
    useState(initialStateValue) {
      stateValue = stateValue || initialStateValue

      function setState(value) {
        stateValue = value
      }

      return [stateValue, setState]
    }
  })
})()

对于 useEffect 相当于结合了 class Component 中的 componentDidMount、componentDitUpdate、componentWillUnmount 这三个声明周期。

也就是说，useEffect 声明的回调函数会在组件挂载、更新、卸载的时候执行。useEffect 的第二个参数是数组类型，只有渲染时数组中的值发生了变化，才会执行该 useEffect 回调。

下面是模拟实现 useEffect 的实现代码：

const React = (function() {
  let deps

  return Object.assign(React, {
    useEffect(callback, depsArray) {
      const shouldUpdate = !depsArray

      const depsChange = deps ? !deps.every((depItem, index) => depItem === depsArray[index]) : true

      if (shouldUpdate || depsChange) {
        callback()

        deps = depsArray || []
      }
    }
  })
})()

useState、useEffect 更多的是在组件内多次调用，那么我们的实现为了区分每次调用 useState 之前不同耳朵 state 值及对应的 setter 函数，就需要额外使用一个数组来存储每次调用的配对值，代码如下：

const React = (function() {
  let hooks = []
  let currentHook = 0

  return Object.assign(React, {
    useState(initialStateValue) {
      hook[currentHook] = hooks[currentHook] || initialStateValue
      function setState(value) {
        hooks[currentHook] = value
      }

      return [hooks[currentHook++], setState]
    },
    useEffect(callback, depsArray) {
      const shouldUpdate = !depsArray
      
      const depsChange = hooks[currentHook] ? !hooks[currentHook].every((depItem, index) => depItem === depsArray[index]) :true

      if (shouldUpdate || depsChange) {
        callback()

        hooks[currentHook++] = depsArray || []
      }
    }
  })
})()

这也是 hook 只可以在顶层使用，不能写在循环体、条件渲染、或者嵌套 function 里的原因。React 内部实现需要按调用顺序来记录每个 useState 的调用，已做区分。

# useReducer 和 Redux

如果 state 的变化有比较复杂的状态流转，则可以使用新的 hook：useReducer 可以让应用更加 Redux 化，使逻辑更加清晰。 使用 useState 有以下几种场景：

  • state 为基本类型。
  • state 转换逻辑简单的场景
  • state 转换只会在当前组件中出现，其他组件不需要感知这个 state
  • 多个 useState hook 之间的 state 并没有关联关系

使用 useReducer 的场景有以下几种：

  • state 为引用类型
  • state 转换逻辑比较复杂的场景
  • 不同 state 之间存在较强的关联关系，应该作为一个 object，用一个 state 来表示的场景。
  • 需要更好的可维护性和可测试下的场景

其实，翻看 React 源码中的 useState 实现就会发现，useState 本质上是 useReducer 的一个语法糖。

我们可以借助 useReducer 实现一个类似 Redux 的数据状态管理。在 store.js 文件中设置 reducer 和数据初始状态：

import React from 'react'
const store = React.createContext(null)

export const initialState = {
  // ...
}

export const reducer = (state, action) => {
  switch(action.type) {
    // ...
  }
}

export default store

接着使用 Provider 来进行根组件挂载，代码如下：

import React, { useReducer } from 'react'
import store, { reduce, initialState } from './store'

function App() {
  const [state, dispatch] = useReducer(reducer, initialState)

  return (
    <store.Provider value={{state, dispatch}}>
      <div/>
    </store>
  )
}

有了 store，业务组件就可以按照下面代码所示的方式来实现状态管理了:

import React, { useContext } from 'react'
import store from './store'

const Child = (props) => {
  const { state, dispatch } = useContext(store)
  // ...
}

React 中能成为 hook 有两个特定条件：

  • 可组合：这个新特性需要具有组合能力，也就是说需要有复用价值，因为 hook 的一大目标就是完成组件的复用。因为，开发者可以自定义 hook，而不必使用官方指定的 hook。
  • 可调式：hook 的一大特性就是能够调试，如果应用出现差错，要能够从错误的 props 和 state 中找到错误的组件或逻辑，具有这样调试功能的特性才有资格成为一个 hook。

React.Suspense 给了 React 组件异步渲染的能力。React.lazy 带来了延迟加载的能力。代码如下：

const Component = React.lazy(() => import(/* webpackChunkName: "Component1" */ './component1'))

export const AsyncComponent = props => {
  <React.Suspense fallback={<Loading>}>
    <Component {...props}>
  <React.Suspense>
}

React.lazy 封装动态 import 的 React 组件，要求 import 必须返回一个 Promise 对象。并且 这个 Promise 对象会被决议为一个 ES 模块，模块中 export default 必须是一个合法的 react 组件。

React.Suspense 组件中设置了 fallback prop，当发现 Component 是一个 Promise 类型，且这个 Promise 没有决议时，就启用 fallback prop 所提供的组件，以便在等待网络返回结果时进行服务器端渲染。

# 路由导航

react-router-dom 的路由导航分为组件式导航和编程式导航；组件式导航就是通过 Link、NavLink 这种方式去进行路由跳转，编程式导航是通过代码控制进行跳转。

这里还需要注意的是，通过 Route 组件的 component 属性渲染出来的组件，他的 props 都会带有 history、location、match 三个对象；可以通过 location 进行编程式子导航，如果没有通过 Route 组件进行渲染出来的组件不能获取该对象，可以通过 Link、NavLink 组件的方式进行跳转；

# 组件式导航

  • 使用 Link 标签进行跳转：
import React from 'react'
import Router from '../../Router'
import { Link } from 'react-router-dom'

const Header = (props: any) => {

  return (
    <div>
      <ul>
        {
          Router.map(item => {
            return (
              <li key={item.name}>
                <Link to={item.path}>{item.name}</Link>
              </li>
            )
          })
        }
      </ul>
    </div>
  )
}

export default Header
  • 路由高亮：使用 NavLink 标签
import React from 'react'
import Router from '../../Router'
import { NavLink } from 'react-router-dom'

const Header = (props: any) => {

  return (
    <div>
      <NavLink activeClassName='active' to='/map'>电影1</NavLink>
      <NavLink to='/mp'>电影2</NavLink>
      <NavLink to='/ap'>电影3</NavLink>
      <NavLink to='/m'>电影4</NavLink>
    </div>
  )
}

export default Header

这里路由高亮的意思是类似单选按钮，路由如果匹配到了某个 NavLink 他会激活 active 类样式，当然这个 active 类需要自己进行添加。

  • 路由重定向： 使用 Redirect 标签，一般配合 Switch 一块使用。
import React from 'react'
import Router from '../../Router'
import { Redirect } from 'react-router-dom'

const Header = (props: any) => {

  return (
    <div>
      <Redirect from='/' to='/home' />
    </div>
  )
}

export default Header

Redirect 标签一样属于普通路由，要放在特殊路由下边；from 和 to 表示当路由匹配到 '/' 时重定向到 '/home'

# 编程式导航

前面说过通过 Route 渲染的组件的 props 都会带有 history、location、match 三个对象：

  • history：主要包括路由跳转函数。
  • location：主要包含路径信息，类似于 Vue-router 的 $route。
  • match：主要使用其中的 params 获取动态路由的信息。

跳转代码：

props.history.push('/map')

import React, { useState } from 'react'
import Header from './component/Header'
import { Route, Switch, NavLink } from 'react-router-dom'
import Map from './component/map'

const App = (props: any) => {
  const onHanleClick = (p: string) => {
    console.log(p)
  }

  return (
    <>
      <Header onHanleClick={onHanleClick}/>
      <Switch>
        <Route path="/p" component={(props: any) => { 
         return (
           <div>{JSON.stringify(props)}
             <button onClick={() => props.history.push('/map')}>tiaozhuan</button>
           </div>
         )
        }}/>
        <Route path="/map" component={Map}/>
      </Switch>
    </>
  )
}

export default App
