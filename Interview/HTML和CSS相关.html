# 语义化

# 语义化是什么

简单说就是，根据结构化的内容选择合适的标签。合适的标签是对内容的高度概括。使用语义化标签的好处如下：

  • 有利于 SEO
  • 开发维护体验更好
  • 用户体验更好（比如，使用 alt 标签来解释图片信息）
  • 更好的可访问性，方便任何设备（如盲人阅读器）对代码进行解析

语义化有一个 Microformats（微格式），是 HTML 标记某些小实体的小模式。他们是在 HTML 中嵌套语义的简单协议，且能迅速地提供一套可被搜索引擎、聚合器等其他工具使用的 API。Microformats 原理就是通过扩展 HTML 元素或者属性，来增强 HTML 的语义表达能力。比如 google 的搜索引擎可以通过 wilipedia 页面中的 vCard 这个 class 读取相关内容，在呈现搜索结果时展现出与搜索内容相关的任务信息。语义化的 class 可以帮助机器学习到更多信息，展现更好的结果页面。

# BFC 布局问题

# BFC 是什么

BFC 是 Block Formatting Context 的简写，翻译过来就是块级格式化上下文。他会创建一个特殊的区域，在这个区域中，只有 block box 参与布局；而 BFC 的一系列特点和规则规定了在这个特殊的区域中如何进行布局，如何进行定位，区域内元素的相互关系和相互作用是怎样的。这个特殊的区域不受外界影响。

block box 是指 display 属性为 block、list-item、table 的元素。相应的 inline box 也是一种 box 类型，是指 display 属性为 inline、inline-block、inline-table 的元素。

# 如何形成 BFC

  • 根元素或其他包含根元素的元素
  • 浮动元素（元素的 float 不是 none）
  • 绝对定位元素（元素的 position 为 absolute 或 field）
  • 内联块（元素具有 display: inline-block 属性）
  • 表格单元格（元素具有 display: inline-cell，HTML 表格单元格默认属性）
  • 表格标题（元素具有 display: table-caption，HTML 表格标题默认属性）
  • 具有 overflow 且值不是 visable 的块元素
  • 含有样式属性 display: flow-root 的元素
  • 含有样式属性 column-span: all 的元素

# BFC 决定了什么

BFC 规则如下：

  • 内部的 box 将会独占宽度，且在垂直方向上一个接一个排列
  • box 在垂直方向上的间距由 margin 属性决定，但是同一个 BFC 的两个相邻 box 的 margin 会出现边距折叠现象
  • 每个 box 在水平方向上的左边缘与 BFC 的左边缘相对齐，即使存在浮动也是如此
  • BFC 区域不会与浮动元素重叠，而是会依次排列
  • BFC 区域是一个独立的渲染容器，容器内的元素和 BFC 区域外的元素质检不会有任何干扰
  • 浮动元素的高度也参与 BFC 的高度计算

从这些规则中，我们至少能总结出如下一些关键要点：

  • 边距折叠
  • 清除浮动
  • 自适应多栏布局

# BFC 实战

如下代码：

<style>
  body {
    width: 600px;
    position: relative;
  }

  .left {
    width: 80px;
    height: 150px;
    float: left;
    background: blue;
  }

  .right {
    height: 200px;
    background: red
  }
</style>

<body>
  <div class="left"></div>
  <div class="right"></div>
</body>

修改代码实现自适应，left 与 right 占满剩下的宽度，进行两栏布局。两个 box 在一个左边缘是因为：每个 box 在水平方向上的左边缘与 BFC 的左边缘相对齐，即使存在浮动也是如此；

BFC 有一个规则：BFC 区域不会与浮动元素重叠，而是会依次排列。我们可以使 right 形成 BFC。可以加一个 overflow: hidden 解决。

下面一个例子：

  <style>
    .root {
      border: 5px solid blue;
      width: 300px;
    }

    .child {
      border: 5px solid red;
      width: 100px;
      height: 100px;
      float: left;
    }
  </style>
  
  <div class="root">
    <div class="child child1"></div>
    <div class="child child2"></div>
  </div>

第一个问题：root 的高度多少；事实上，因为 child 为浮动元素，因此会造成高度塌陷现象。 root 的高度为 0；根据 BFC 的规则，浮动元素的高度也参与 BFC 的高度计算，因此可以使 root 形成 BFC 来解决问题。添加 overflow: hidden 来清除浮动；

下面例子：

  <style>
    p {
      color: aqua;
      background: rgb(216, 148, 148);
      width: 400px;
      line-height: 100px;
      text-align: center;
      margin: 40px;
    }
  </style>
  <p>paragraph1</p>
  <p>paragraph2</p>

问题：两段之间的垂直距离为多少。根据 BFC 布局规则，box 在垂直方向上的间距由 margin 属性决定，但是同一个 BFC 的两个相邻 box 的 margin 会出现边距折叠现象，所以该问题答案为 40px；解决这个问题，最简单的就是可以在 p 标签中再包裹一个元素，并触发该元素形成一个 BFC，那么这两个 p 标签就不在属于同一个 BFC 了，这样就可以解决问题。

代码如下：

  <style>
    p {
      color: aqua;
      background: rgb(216, 148, 148);
      width: 400px;
      line-height: 100px;
      text-align: center;
      margin: 40px;
    }
    .wrapper {
      overflow: hidden;
    }
  </style>
  <p>paragraph1</p>
  <div class="wrapper">
    <p>paragraph2</p>
  </div>

# 通过多种方式实现居中

例子：

  <style>
    .wp {
      border: 1px solid rebeccapurple;
      width: 300px;
      height: 300px;
    }

    .box {
      background-color: aquamarine;
    }

    .box .fixed-size {
      width: 100px;
      height: 100px;
    }
  </style>
  <div class="wp">
    <div class="box fixed-size">text</div>
  </div>

下面主要从居中元素定宽高、居中元素不定宽高两个层面来介绍实现居中显示的不同方法；

# 居中元素定宽高

主要有以下三种：

  1. absolute + 负 margin

代码如下：

.wp {
  position: relative;
}

.box {
  position: absolute;
  top: 50%;
  left: 50%;
  margin-left: -50%;
  margin-right: -50%;
}
  1. absolute + margin auto

这种方法是将各个方向的距离都设置为 0，此时将 margin 配置为 auto，实现居中。

    .wp {
      width: 300px;
      height: 300px;
      background: #ddd;
      position: relative;
    }

    .box {
      width: 200px;
      height: 100px;
      background: orange;
      position: absolute;
      top: 0;
      bottom: 0;
      left: 0;
      right: 0;
      margin: auto;
      text-align: center;
      line-height: 100px;
    }
  1. absolute + calc

跟第一种类似：

    .wp {
      width: 300px;
      height: 300px;
      background: #ddd;
      position: relative;
    }

    .box {
      width: 200px;
      height: 100px;
      background: orange;
      position: absolute;
      top: calc(50% - 50px);
      left: calc(50% - 100px);
    }

# 居中元素不定宽高

例子如下：

  <style>
    .wp {
      width: 300px;
      height: 300px;
      background: #ddd;
    }

    .box {
      background: orange;
    }
  </style>

<body>
  <div class="wp">
    <div class="box">text</div>
  </div>
</body>
  1. absolute + transform

宽高不定时，可以利用 css3 的 transform，transform 的 translate 属性也可以用来设置百分比。这个百分比是基于自身的宽和高计算出来的，因此可以将 translate 设置为 50%；

    .wp {
      position: relative;
      width: 300px;
      height: 300px;
      background: #ddd;
    }

    .box {
      position: absolute;
      top: 50%;
      left: 50%;
      transform: translate(-50%, -50%);
      background: orange;
    }
  1. lineheight

将 box 元素设置为行内元素，可以通过 text-align 实现水平居中，同时通过 vertical-align 实现垂直方向上的居中：

    .wp {
      width: 300px;
      height: 300px;
      line-height: 300px;
      text-align: center;
      font-size: 0px;
      background: #ddd;
    }

    .box {
      font-size: 16px;
      display: inline-block;
      vertical-align: middle;
      line-height: initial;
      text-align: left;
      /* 修正文字 */
      background: orange;
    }
  1. table

代码如下：

  <style>
    .wp {
      width: 300px;
      height: 300px;
      text-align: center;
      background: #ddd;
    }

    .box {
      display: inline-block;
      background: orange;
    }
  </style>

<body>
  <table>
    <tbody>
      <tr>
        <td class="wp">
          <div class="box">text</div>
        </td>
      </tr>
    </tbody>
  </table>
</body>
  1. css-table

如何使用 table 布局的特性效果，但是不采用 table 元素呢，答案是使用 css-table，代码如下：

    .wp {
      width: 300px;
      height: 300px;
      display: table-cell;
      text-align: center;
      vertical-align: middle;
      background: #ddd;
    }

    .box {
      display: inline-block;
      background: orange;
    }
  1. flex
    .wp {
      width: 300px;
      height: 300px;
      display: flex;
      justify-content: center;
      align-items: center;
      background: #ddd;
    }

    .box {
      background: orange;
    }
  1. grid
    .wp {
      width: 300px;
      height: 300px;
      display: grid;
      background: #ddd;
    }

    .box {
      align-self: center;
      justify-self: center;
      background: orange;
    }

对布局总结：

  • PC 端有兼容性要求，宽高固定，推荐使用 absolute + 负 margin 的方法来实现居中
  • PC 端有兼容性要求，宽高不固定，推荐使用 css-table 实现居中
  • PC 端无兼容性要求，推荐使用 flex 布局
  • 移动端推荐使用 flex 布局

# Web components

优点如下：

  • 原生规范，无须使用框架

他是继承了且具备了框架的优点；新的 web components 规范中新增了组件声明周期、slot、模板等概念（可以与 JSX 或 Vue template 类比），这些概念与本来就已经存在的组件化、shadow dom、扩展原生元素的能力相结合，使 web components 具备了较好的发展前景。

  • 原生使用，无须进行编译

web components 是原生工具，会得到浏览器的天然支持，可以免去编译构建过程。

  • 真正的 CSS scope

web components 实现了真正的 CSS scope，做到了样式隔离。

# 移动端 H5 注意事项总结

  1. 打电话、发短信、写邮件的小技巧

<a href="tel: 110">110<a>

发短信：

<a href="sms: 110">110<a>

写邮件，实现此功能需要依赖 mailto

<a href="mailto: 110@govn.com; jiegiser@163.com">110<a>

<a href="mailto: 110@govn.com?subject=SOS">110<a>

使用 body 自定义邮件内容：

<a href="mailto: 110@govn.com?subject=SOS&body=SOS">110<a>

这是由于历史原因造成的，一般解决手段为禁止混用 touch 和 click，或者增加一层 “透明” 蒙层，也可以通过延迟上层元素消失来实现。

在移动端某些设备上，点击按钮会产生丑陋的边框，可以使用下面代码，对默认样式进行禁用：

-webkit-tap-highlight-color: rgba(0, 0, 0, 0);
  1. 长按链接与图片时禁止弹出菜单
-webkit-touch-callout: none;
  1. 禁止用户选中文字
-webkit-user-select: none;
user-select: none;
  1. 取消输入英文时默认首字母大写

在 IOS 下生效：

# CSS 变量和主题切换优雅实现

# CSS 变量

比如下面代码：

body {
  background: white;
  color: #555;
}
a, a:link {
  color: #639A45;
}
a:hover {
  color:#205D65;
}

借助 CSS 变量，我们可以对颜色色值进行设置，如下：

:root{
  --bg: white;
  --text-color: #555;
  --link-color: #639A45;
  --link-hover: #205D65;
}

简化代码，如下：

body {
  background: var(--bg);
  color: var(--text-color);
}
a, a:link {
  color: var(--link-color);
}
a:hover {
  color:var(--link-hover);
}

声明的变量名：--变量名，然后使用 var 去消费变量。必须使用 --变量名，去定义变量。

# 使用 CSS 变量实现主题切换

如下代码：

:root {
  --bg: white;
  --text-color: #555;
  --link-color: #639A45;
  --link-hover: #205D65;
}

定义一个 .pink-theme 来对应粉色主题，代码如下：

:pink-theme {
  --bg: hotpink;
  --text-color: white;
  --link-color: #639A45;
  --link-hover: #205D65;
}

这样一来，切换主题就变得简单了，代码如下：

const toggleaBtn = document.querySelector('.toggle-theme')

toggleB.addEventListener('click', e => {
  e.preventDefault()

  if (document.body.classList.contains('pink-theme')) {
    // 当前主题为粉色主题，需要移除 pink-theme class
    document.body.classList.remove('pink-theme')

    toggle.innerText = '切换正常主题'
  } else {
    document.body.classList.add('pink-theme')
    toggle.innerText = '切换为粉色主题'
  }
})

也可以利用 localStorage 实现主题的保存，代码如下：

const toggleaBtn = document.querySelector('.toggle-theme')

if (localStorage.getItem('pickTheme')) {
  document.body.classList.add('pink-theme')
  toggle.innerText = '切换为粉色主题'
}

toggleB.addEventListener('click', e => {
  e.preventDefault()

  if (document.body.classList.contains('pink-theme')) {
    // 当前主题为粉色主题，需要移除 pink-theme class
    document.body.classList.remove('pink-theme')

    toggle.innerText = '切换正常主题'
    localStorage.removeItem('pinkTheme')
  } else {
    document.body.classList.add('pink-theme')
    toggle.innerText = '切换为粉色主题'
    localStorage.setItem('pinkTheme', true)
  }
})

# CSS Modules

css Modules 就是类似 vue 的 scope 的作用；让样式在局部起作用。

使用 webpack 中配置 css-loader 设置 module 参数即可进行 CSS Modules 处理。

在通过 create-react-app 脚手架构建的 react 项目，是打开了 CSS Modules 处理的，需要在命名 css 文件的时候，以 文件名.module.后缀，这样才可以。
