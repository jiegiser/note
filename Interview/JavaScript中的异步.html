# 异步——红绿灯任务控制

一个异步题： 红灯 3s 亮一次，绿灯 1s 亮一次，黄灯 2s 亮一次，如何让 3 个灯不断交替重复亮呢？已知 3 个亮灯函数已经存在，如下：

function red() {
  console.log('red')
}

function green() {
  console.log('green')
}

function yellow() {
  console.log('yellow')
}

# callback 方案

代码如下：

const task = (timer, light, callback) => {
  setTimeout(() => {
    if (light === 'red') {
      red()
    } else if (light === 'green') {
      green()
    } else if (light === 'yellow') {
      yellow()
    }
    callback()
  }, timer)
}

const step = () => {
  task(3000, 'red', () => {
    task(1000, 'green', () => {
      task(2000, 'yellow', step)
    })
  })
}

# Promise 解决

代码：

const task = (timer, light) => {
  new Promise((resolve, reject) => {
    setTimeout(() => {
      if (light === 'red') {
        red()
      } else if (light === 'green') {
        green()
      } else if (light === 'yellow') {
        yellow()
      }
      resolve()
    }, timer)
  })
}

const step = () => {
  task(3000, 'red')
  .then(() => task(1000, 'green'))
  .then(() => task(2000, 'yellow'))
  .then(step)
}

# async/await 解决

代码：

const taskRunner = async () => {
  await task(3000, 'red')
  await task(1000, 'green')
  await task(2000, 'yellow')
  taskRunner()
}

taskRunner()

# 请求图片进行预先加载

假设 urlids 数组预先就存在，数组中的每一项都可以按照规则拼接成一个完整的图片地址，那么请根据这个数组，依次请求图片进行预加载。

使用 Promise 实现：

const loadImg = urlId => {
  const url = `http://www.image.com/${urlId}`
  return new Promise((resolve, reject) => {
    const img = new Image()
    img.onerror = function () {
      reject(urlId)
    }

    img.onload = function () {
      resolve(urlId)
    }
    img.src = url
  })
}

// 根据 urlId 依次请求图片，代码如下：
const urlId = [1, 2, 3, 4, 5]

urlId.reduce((prevPromise, urlId) => {
  return prevPromise.then(() => loadImg(urlId))
}, Promise.resolve())

上面使用了数组的 reduce 方法，也可以使用面向过程的方法来实现，代码如下：

const loadImgOneByOne = index => {
  const length = urlId.length
  loadImg(urlIds[index]).then(() => {
    if (index === length - 1) {
      return
    } else {
      loadImgOneByOne(++index)
    }
  })
}

还可以使用 async/await 来实现。代码如下：

const loadImgOneByOne = async () => {
  for (i of urlIds) {
    await loadImg(urlIdsp[i])
  }
}

loadImgOneByOne()

如果需要提高效率，将所有图片的请求一次性发出，可以使用 Promise.all 来实现，代码如下：

const urlIds = [1, 2, 3, 4, 5]

const promiseArray = urlIds.map(urlIf => loadImg(urlId))

Promise.all(promiseArray).then(() => {
  console.log('finish load all')
}).catch(() => {
  console.log('promise all catch')
})

如果我们希望控制最大并发数为3，最多一起发出3个请求，剩下2个一起发出。这时候就需要实现一个 loadByLimit 方法，实现时可以考虑使用 Promise.race 方法，代码如下：

const loadByLimit = (urlIds, loadImg, limit) => {
  const urlIdsCopy = [...urlIds]

  if (urlIdsCopy.length <= limit) {
    // 如果数组长度小于最大并发数，直接发出全部请求
    const promiseArray = urlIds.map(urlId => loadImg(urlId))
    return Promise.all(promiseArray)
  }

  // 注意，splice 方法会改变 urlIdsCopy 数组
  const promiseArray = urlIdsCopy.splice(0, limit).map(urlId => loadImg(urlId))

  promiseArray.reduce((prevPromise, urlId) => {
    prevPromise
    .then(() => {
      Promise.race(promiseArray)
    })
    .catch(error => { console.log(error) })
    .then(resolveId => {
      // 将 resolvedId 从 promiseArray 数组中删除
      // 这里用于删除操作的知识伪代码，具体删除情况要看后端 API 返回的结果
      let resolvedIdPosition = promiseArray.findIndex(id => resolveId === id)
      promiseArray.splice(resolvedIdPosition, 1)
      promiseArray.push(loadImg(urlId))
    })
  }, Promise.resolve()).then(() => {
    Promise.all(promiseArray)
  })
}

# setTImeout 相关考察

JavaScript 中任务分为同步任务和异步任务；

  • 同步任务：当前主线程将要消化执行的任务，这些任务一起形成执行栈；
  • 异步任务：不进入主线程，而是进入任务队列，即不会马上进行的任务；

当同步任务全部都被消化，主线程空闲时，即上面提到的执行栈为空时，系统将会执行任务队列中的任务，即异步任务。

这样的机制保证了：虽然 JavaScript 是单线程的，但是对于一些耗时的任务，我们可以将其丢如任务队列中，这样就不会阻碍其他同步代码的执行，等到异步任务完成后，便会进行相关逻辑的操作。

关于 setTimeout，如下面例子：

setTimeout(() => {
  console.log('here 100')
}, 100)

setTimeout(() => {
  console.log('here 2')
}, 0)

这段代码，以为第二个 setTimeout 的定时器函数将更快被触发，所以先加入到异步队列中，被执行。但是如果修改代码如下：

setTimeout(() => {
  console.log('here 1')
}, 1)

setTimeout(() => {
  console.log('here 2')
}, 0)

上面代码执行就不一样了，在 Chrome 浏览器中先是第一个 setTimeout，然后是第二个。这是因为在 1ms 以内的定时都以最小延迟时间处理。谁的代码靠前，谁就先执行。

mdn 上给出的最小延迟时间为 4ms，另外 setTimeout 也有最大延迟时间的概念，这都依赖于规范的指定和浏览器引擎的实现。

# 宏任务和微任务

首先看下面代码：

console.log('start here')

new Promise((resolve, reject) => {
  console.log('first promise constructor')
  resolve()
}).then(() => {
  console.log('first promise then')
  return new Promise((resolve, reject) => {
    console.log('second promise')
    resolve()
  }).then(() => {
    console.log('second promise then')
  })
}).then(() => {
  console.log('another first promise then')
})

console.log('end here')

// start here
// first promise constructor
// end here
// first promise then
// second promise
// second promise then
// another first promise then

事实上，Promise 的完成处理函数也会被放入任务队列中，但是这个任务队列和前面所提到的与 setTimeout 相关的任务队列有所不同。

任务队列中的异步任务其实又分为宏任务与微任务，也就是说宏任务和微任务虽然都是异步任务，都在任务队列中，但是他们在两个不同的队列中。

如何区分宏任务与微任务，一般情况下，宏任务包括下面内容：

  • setTimeout
  • setInterval
  • I/O
  • 事件
  • postMessage
  • setImmediate（NodeJS 中的特性，浏览器端已经废弃该 API）
  • requestAnimationFrame
  • UI 渲染

微任务包括：

  • Promise.then
  • MutationObverse
  • process.nextTrick（NodeJS）

对于代码中同时存在宏任务以及微任务，谁的优先级更高，可以先看下面代码：

console.log('start here')

const foo = () => {
  return new Promise((resolve, reject) => {
    console.log('first promise constructor')

    let promise1 = new Promise((resolve, reject) => {
      console.log('second promise constructor')

      setTimeout(() => {
        console.log('setTimeout here')
        resolve()
      })
      resolve('promise1')
    })

    promise1.then(arg => {
      console.log(arg)
    })
  })
}

foo().then(arg => {
  console.log(arg, 'arg')
})

console.log('end here')
// start here
// first promise constructor
// second promise constructor
// end here
// promise1
// setTimeout here

看上面代码可以知道，虽然 setTimeout 回调函数先进入任务队列，但是引擎会优先执行微任务，按照微任务的顺序先输出 promise1 的结果，再输出 promise0 的结果（即 第一个匿名 promise 的结果）。

由上分析可知，每次主线程执行栈为空的时候，引擎都会优先处理微任务队列，处理完微任务队列中的所有任务，再处理宏任务。如同下面代码输出结果一般：

console.log('start here')

setTimeout(() => {
  console.log('setTimeout')
}, 0)

new Promise((resolve, reject) => {
  resolve('promise result')
}).then(value => console.log(value))

console.log('end here')

// start here
// end here
// promise result
// setTimeout
